# Paperclip CEO 面板 -- 布局骨架设计文档

> 目标：将现有"三栏聊天"demo 重构为"左侧导航 + 主工作台 + 右侧沟通"结构。
> 本文档只描述设计意图与交互规格，不包含实现代码。

---

## 1. 整体布局网格

### 1.1 顶层结构

页面保持现有的两层网格：顶层 `app` 容器维持 `grid-template-rows: 52px 1fr`，即 52px 顶栏 + 弹性主体。顶栏不变，仍承载品牌标识、公司状态和主题切换。

主体区域 `shell` 从当前的三等分列改为以下三列布局：

```
grid-template-columns: 56px 1fr 360px;
```

三列依次为：

| 列序 | 名称 | 宽度 | 说明 |
|------|------|------|------|
| A | 功能导航栏 | 56px 固定 | 纯图标 + 文字标签的紧凑导航 |
| B | 主工作台 | 1fr 弹性填充 | Issue、Approval、Agent 等功能的工作区 |
| C | 沟通侧栏 | 360px 固定 | 聊天、员工列表、任务指派入口 |

### 1.2 区域命名（grid-template-areas）

```
"nav  workspace  comm"
"nav  workspace  comm"
```

所有三行共用同一列定义，纵向由各区域内部自行管理滚动。

### 1.3 列间距

列之间用 `1px solid var(--border)` 分隔线隔开，与现有 demo 的 `border-right` 分隔方式一致。不使用 gap，以避免与现有视觉语言冲突。

## 2. 各区域内容与职责

### 2.1 区域 A：功能导航栏（56px）

**定位**：全局功能入口，不含业务内容，只提供导航跳转和当前模块指示。

**内容**（从上到下）：

1. **品牌缩略图标** -- 使用现有 Paperclip 回形针 SVG 的简化版，16x16 或 20x20。位于导航栏顶部，与顶栏品牌区视觉关联。
2. **导航项目列表**：
   - Issue（任务）-- 默认激活项
   - Approval（审批）
   - Agent（员工/代理）
   - Activity（活动日志）
3. **底部固定项**：
   - 设置（齿轮图标，预留入口）
4. **当前激活指示**：激活项左侧或右侧有 3px 宽的 `var(--accent)` 色条，与现有 `.item-btn.active` 的 `border-left` 指示器风格一致。

**视觉规格**：

- 每个导航项高度 44px，水平排列为"上方图标 + 下方文字标签"（10px 字号）。
- 整栏背景 `var(--panel)`，hover 状态 `var(--muted)`。
- 不设 panel-head / panel-foot 的上下内边距区域，保持极简。
- 使用 `position: sticky; top: 0` 确保导航不随主体滚动。

**交互规则**：

- 点击导航项切换主工作台内容（参见第 4 节状态管理）。
- 不直接切换右侧沟通侧栏。
- 无右键菜单，无子菜单，保持一级扁平结构。

### 2.2 区域 B：主工作台（弹性填充）

**定位**：承载所有业务操作的核心区域，根据左侧导航选择展示不同模块。

#### 2.2.1 通用结构

每个模块共享以下骨架：

```
模块标题栏（module-head）
  -- 模块名称（h1, 16px, font-weight 600）
  -- 模块描述（12px, var(--muted-fg)）
  -- 操作按钮区（如"新建 Issue"、"批量审批"）

模块工具栏（module-toolbar, 可选）
  -- 筛选器、搜索框、视图切换（列表/看板）

模块内容区（module-body）
  -- 滚动容器，min-height: 0, overflow: auto
```

#### 2.2.2 Issue 模块（默认首页）

**标题栏**：Issue 工作台 + "新建 Issue" 主操作按钮。

**工具栏**：
- 状态筛选：全部 / 进行中 / 已完成 / 已暂停
- 优先级筛选：高 / 中 / 低
- 指派人筛选：下拉选择 Agent
- 搜索框

**内容区 -- 列表视图**：
- 每行一个 Issue，显示：标题、状态标签（pill）、优先级色条、指派人头像缩略、截止时间。
- 行高约 56px，hover 时 `var(--muted)` 背景。
- 点击 Issue 行展开详情面板（覆盖当前列表，提供"返回列表"操作），详情面板内包含：
  - 标题、描述、状态变更历史
  - 指派信息（Agent、上级 Issue）
  - 操作按钮：编辑、状态流转、添加评论
  - 关联 Approval 链接（如有）

**内容区 -- 看板视图**（可选增强）：
- 按 Issue 状态分列（待处理 / 进行中 / 待审批 / 已完成）。
- 每列内卡片垂直排列，卡片内容精简：标题 + 指派人 + 优先级。

#### 2.2.3 Approval 模块

**标题栏**：审批队列 + "我的待审批" 数量徽标。

**工具栏**：
- 筛选：待我审批 / 我发起的 / 全部
- 类型筛选：招聘 / 战略 / 预算 / 其他

**内容区**：
- 卡片式列表，每张卡片包含：
  - 审批类型标签（pill，复用现有 `.badge` 样式体系）
  - 发起人、发起时间
  - 摘要描述
  - 操作按钮组：批准 / 驳回 / 详情
- "待我审批"项排在最前，带视觉强调（左侧色条或浅底色）。
- 批准操作提供 inline 确认（不弹模态框），确认后卡片标记为已处理并灰度显示。

#### 2.2.4 Agent 模块

**标题栏**：Agent 管理 + "招聘新 Agent" 按钮（标记为需董事会审批的提示）。

**工具栏**：
- 搜索框（复用现有 `.employee-search` 样式）
- 状态筛选：在线 / 忙碌 / 离线 / 全部

**内容区**：
- 沿用现有 `.employee-card` 卡片样式，但增加更多信息：
  - 头像、姓名、角色、部门（现有）
  - 当前负责 Issue 数量
  - 预算状态指示器（正常/警告/冻结）
- 点击卡片展开 Agent 详情面板，包含：
  - 基本信息、上级 Agent
  - 当前 Issue 列表（可跳转到 Issue 模块）
  - 预算消耗进度
  - 操作：编辑、冻结/解冻预算、发起单聊（跳转到右侧沟通侧栏）

#### 2.2.5 Activity 模块

**标题栏**：活动日志 + 时间范围选择器。

**工具栏**：
- 实体类型筛选：Issue / Approval / Agent / System
- 搜索

**内容区**：
- 时间线式列表，每条记录显示：
  - 时间戳（monospace，复用 `.time` 样式）
  - 操作者（Agent 头像 + 名称）
  - 动作描述（如"将 Issue #12 状态变更为 进行中"）
  - 关联实体链接（可跳转到对应模块详情）
- 按日期分组，复用现有 `.divider` 分隔线样式。

### 2.3 区域 C：沟通侧栏（360px）

**定位**：将现有 demo 的全部聊天能力整合到可折叠侧栏中，作为辅助沟通通道而非主工作区。

#### 2.3.1 侧栏折叠机制

- 侧栏默认展开（360px）。
- 通过顶栏或侧栏边缘的折叠按钮可收起为仅显示宽度 0 的隐藏状态（或 48px 的窄条，仅显示未读指示器）。
- 折叠/展开使用 CSS transition，过渡时长 200ms，缓动函数 `ease-out`。
- 折叠状态下，主工作台自动扩展填充右侧空间（`grid-template-columns: 56px 1fr 0px` 或通过 JS 切换类名控制）。

#### 2.3.2 侧栏内部结构

侧栏内部采用子网格或 flex 列布局，从上到下分为：

**C1. 会话切换栏（固定高度，约 48px）**
- 水平 tab 式切换：总群 / 公告 / 搜索（对应员工单聊入口）。
- 复用现有 `sessions` 数据中的 `all-hands` 和 `announcements`。
- "搜索" tab 点击后展开员工搜索框（复用现有 `.employee-search`），搜索结果直接列出匹配员工，点击即打开单聊。

**C2. 消息区域（flex: 1, overflow: auto）**
- 完全复用现有 `.messages` 容器的消息渲染逻辑：
  - 消息气泡（头像、作者、badge、时间戳、正文）
  - 系统提醒条（`.notice`）
  - 任务草稿条（`.task-strip`）
  - 日期分隔线（`.divider`）

**C3. Composer 区域（固定高度，弹性伸缩）**
- 完全复用现有 `.composer` 结构：
  - textarea + 发送按钮
  - composer-meta 提示行和标签
  - 支持"普通消息"和"任务草稿"两种模式
- 任务指派入口（现有"指派任务"按钮）迁移到 Composer 上方或内部的工具栏：
  - 当会话类型为 DM 时，composer 工具栏出现"切换为任务草稿"图标按钮。
  - 点击后进入 `mode: 'assign'`，composer textarea 内容和行为与现有逻辑一致。

#### 2.3.3 员工列表整合方式

现有 demo 中员工列表是独立的中栏面板。重构后：

- 员工列表不再作为常驻面板存在。
- 替代方案：在沟通侧栏的"搜索" tab 下展示完整员工列表（可滚动），每项点击后：
  - 自动创建/定位到该员工的 DM 会话
  - 切换消息区域和 composer 到该 DM
- 员工卡片保留现有样式（头像、姓名、角色、状态点），仅增加一行当前负责 Issue 数量的简要信息。

## 3. 关键交互流程

### 3.1 查看 Issue

1. 用户点击左侧导航栏 Issue 图标。
2. 主工作台切换到 Issue 模块，展示 Issue 列表（默认"全部"状态）。
3. 用户可使用工具栏筛选条件缩小范围。
4. 点击某 Issue 行，主工作台内容从列表切换为该 Issue 的详情面板（URL 或状态记录当前查看的 Issue ID）。
5. 详情面板顶部有"返回列表"按钮。
6. 如果 Issue 关联了 Approval，详情面板内显示链接，点击可跳转到 Approval 模块并定位到该审批。

### 3.2 审批操作

1. 用户点击左侧导航栏 Approval 图标。
2. 主工作台展示审批队列，"待我审批"项优先排列。
3. 用户点击某审批卡片查看详情（可 inline 展开，不跳转页面）。
4. 点击"批准"按钮：
   - 按钮变为确认态（文字变为"确认批准"，背景色短暂变为 `var(--good-soft)`）。
   - 再次点击确认，审批状态更新。
   - 卡片标记为已处理，视觉上灰度化并下沉到列表底部。
5. 点击"驳回"按钮：
   - 弹出 inline 文本框要求填写驳回理由。
   - 提交后卡片同样标记为已处理。
6. 审批结果同步写入 Activity 日志。

### 3.3 打开/关闭沟通侧栏

1. 顶栏右侧新增"沟通"图标按钮（气泡图标）。
2. 点击后切换沟通侧栏的展开/折叠状态。
3. 展开时侧栏从右侧滑入（200ms ease-out），主工作台宽度相应收缩。
4. 折叠时侧栏滑出，主工作台扩展填充。
5. 侧栏折叠状态下，顶栏"沟通"按钮旁显示未读消息数红色徽标（如有新消息）。

### 3.4 发起单聊

1. 用户展开沟通侧栏。
2. 点击会话切换栏的"搜索" tab（或直接点击搜索图标）。
3. 搜索框激活，输入员工姓名或职位关键词。
4. 匹配员工以列表形式展示（复用现有 `.employee-card` 样式）。
5. 点击某员工：
   - 系统自动调用 `ensureDm` 创建/定位 DM 会话。
   - 会话切换栏高亮当前 DM（tab 样式或面包屑回退）。
   - 消息区域切换到该 DM 的消息列表。
   - Composer 切换到单聊模式。
6. 用户在 composer 输入消息并发送。

### 3.5 任务指派

1. 用户在沟通侧栏中已打开某员工的 DM 会话。
2. 点击 composer 上方的"任务草稿"图标按钮（或侧栏顶部的"指派任务"按钮）。
3. Composer 进入 `mode: 'assign'`：
   - textarea 填充任务模板（目标、完成定义、截止时间）。
   - composer-meta 显示"任务草稿"标签和目标员工标签。
4. 用户编辑草稿内容。
5. 点击发送：
   - 消息以"任务草稿"类型追加到当前 DM 会话。
   - 系统自动追加一条系统提醒（复用现有 System bot 消息逻辑）。
   - Composer 恢复普通消息模式。
6. 提示用户：正式指派需在 Issue 工作台中提交确认。

## 4. 状态与导航关系

### 4.1 全局状态模型

```
{
  activeModule: 'issue' | 'approval' | 'agent' | 'activity',
  moduleState: {
    issue: { view: 'list' | 'detail', selectedIssueId: string | null, filters: {...} },
    approval: { filter: 'pending' | 'mine' | 'all', selectedId: string | null },
    agent: { search: string, statusFilter: string, selectedId: string | null },
    activity: { entityType: string, search: string }
  },
  commSidebar: { open: boolean, activeTab: 'group' | 'announcements' | 'search', activeSessionId: string, mode: 'chat' | 'assign' },
  theme: 'light' | 'dark'
}
```

### 4.2 导航切换规则

| 导航项 | 主工作台内容 | 沟通侧栏影响 |
|--------|-------------|-------------|
| Issue | Issue 列表/详情 | 不变 |
| Approval | 审批队列 | 不变 |
| Agent | Agent 列表/详情 | 点击 Agent 详情中"发起单聊"会打开侧栏并切换到该员工 DM |
| Activity | 活动日志 | 点击日志中的实体链接会切换主工作台到对应模块 |

### 4.3 跨模块联动

- Issue 详情中引用 Approval -> 点击链接切换到 Approval 模块并定位。
- Approval 详情中引用 Issue -> 点击链接切换到 Issue 模块并定位。
- Agent 详情中"当前 Issue"列表 -> 点击跳转到 Issue 模块详情。
- Agent 详情中"发起单聊" -> 打开沟通侧栏，定位到该员工 DM。
- Activity 日志中的实体链接 -> 切换到对应模块并定位。
- 沟通侧栏中的"任务草稿"发送后 -> 提示用户到 Issue 工作台正式提交。

### 4.4 侧栏状态持久性

- 侧栏的展开/折叠状态在当前会话内持久（存入 `state.commSidebar.open`）。
- 侧栏内当前激活的会话 ID 和 tab 在导航切换时保持不变，不会因为点击左侧导航而重置。
- 唯一例外：从 Agent 详情点击"发起单聊"时，侧栏自动展开并切换到目标 DM。

## 5. 响应式降级策略

### 5.1 断点定义

| 断点名称 | 宽度范围 | 布局策略 |
|----------|---------|---------|
| 全宽 | > 1280px | 三栏完整展示：56px 导航 + 弹性工作台 + 360px 沟通侧栏 |
| 中宽 | 960px - 1280px | 导航栏 56px 不变；沟通侧栏默认折叠，通过按钮展开（展开时覆盖在主工作台之上，而非挤压） |
| 窄屏 | 600px - 960px | 导航栏收缩为底部 tab 栏（固定在视口底部）；主工作台占满宽度；沟通侧栏以全屏抽屉形式打开 |
| 手机 | < 600px | 同窄屏，但导航栏 tab 项减少为图标（无文字标签）；模块内工具栏折叠为单行 |

### 5.2 降级细节

**中宽断点（960-1280px）**：

- 沟通侧栏不再占据网格列，改为 `position: fixed; right: 0; top: 52px; bottom: 0; width: 360px; z-index: 100`。
- 展开时侧栏从右侧滑入，覆盖主工作台右侧 360px 区域。
- 主工作台始终占满 `calc(100vw - 56px)` 的宽度。
- 侧栏背景加半透明遮罩层（`rgba(0,0,0,.12)`），点击遮罩可关闭侧栏。

**窄屏断点（600-960px）**：

- 左侧导航栏从垂直侧边栏变为水平底部 tab 栏：
  - 高度 52px，宽度 100vw。
  - 导航项水平排列，图标 + 文字标签。
  - 使用 `position: fixed; bottom: 0` 固定在底部。
- 主工作台区域调整为 `height: calc(100dvh - 52px - 52px)`（减去顶栏和底部导航栏高度）。
- 沟通侧栏变为全屏抽屉（`width: 100vw`），从右侧滑入覆盖整个内容区。

**手机断点（<600px）**：

- 底部导航栏高度维持 52px，但导航项只显示图标，文字标签省略。
- 模块内工具栏折叠：筛选条件收入一个"筛选"下拉按钮内，不横向展开。
- Issue 列表行高缩小至 48px，隐藏次要信息（如截止时间），点击后全屏展示详情。
- 沟通侧栏同样为全屏抽屉，composer 区域高度增加（min-height: 100px），方便移动端输入。

### 5.3 暗色模式

- 暗色模式逻辑完全保留，通过 `.dark` 类切换 CSS 变量。
- 新增的导航栏、模块标题栏等所有新增区域一律使用现有变量体系（`var(--bg)`, `var(--fg)`, `var(--panel)`, `var(--border)`, `var(--muted)`, `var(--accent)` 等），不引入新变量。
- 沟通侧栏的覆盖遮罩在暗色模式下使用 `rgba(0,0,0,.4)`。

## 附录 A：现有设计语言保留清单

以下元素必须在新布局中保持不变，不因重构而丢失：

- 品牌标识（回形针 SVG + "Paperclip" 文字）
- 主题切换（明/暗模式，localStorage 持久化）
- 消息渲染体系（头像、badge、时间戳、正文排版）
- 系统提醒条（`.notice`）和任务草稿条（`.task-strip`）
- Composer 双模式（普通消息 / 任务草稿）
- 员工卡片样式（头像、状态点、角色信息）
- CSS 变量体系（不新增、不重命名、不改值）
- 无障碍：键盘导航、focus 状态、ARIA 标签
- 滚动条暗色模式自定义样式

## 附录 B：新增视觉元素清单

以下元素为重构新增，需保持与现有设计语言一致：

| 元素 | 复用策略 |
|------|---------|
